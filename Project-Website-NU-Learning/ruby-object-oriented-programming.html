Object Oriented Programming

Introduction

Welcome to this lesson on NU-Learning, where we will explore the concept of Object Oriented Programming (OOP) in Ruby. OOP is a powerful programming paradigm that allows developers to create applications using objects, which encapsulate data and behavior. By mastering OOP, you can improve your coding skills and write more organized, maintainable Ruby programs.

What is Object Oriented Programming?

Object Oriented Programming (OOP) is a programming style that uses "objects" to design software. Objects are instances of classes, which are blueprints that define attributes and methods. OOP allows for more structured and modular code, making it easier to manage and understand complex systems.

Examples of objects in Ruby could include:

  • A User object representing a user in a system.
  • A Car object that holds properties like color, model, and methods like drive and stop.

Key Principles of OOP

The four main principles of Object Oriented Programming are:

  • Encapsulation: Bundling the data (attributes) and methods that operate on the data into a single unit, or object. This helps protect the object's integrity by restricting direct access to some of its components.
  • Abstraction: Hiding complex implementation details and showing only the essential features of the object. This simplifies the interface for users of the object.
  • Inheritance: Allowing a new class to inherit attributes and methods from an existing class, promoting code reuse.
  • Polymorphism: Allowing objects of different classes to be treated as objects of a common superclass. This enables one interface to control access to a general class of actions.

Creating Classes and Objects

In Ruby, creating a class is straightforward. Here’s how you can define a simple class and create an object:

# Defining a class
class Car
  attr_accessor :color, :model

  def initialize(color, model)
    @color = color
    @model = model
  end

  def drive
    puts "The #{@color} #{@model} is driving."
  end
end

# Creating an object of the class
my_car = Car.new("red", "Toyota")
my_car.drive # Output: The red Toyota is driving.

Instance Variables and Methods

Instance variables hold data specific to each instance of a class. They are defined with a '@' symbol. Methods defined within a class allow objects to perform actions. Here’s an example:

# Adding more methods to the Car class
class Car
  attr_accessor :color, :model

  def initialize(color, model)
    @color = color
    @model = model
  end

  def drive
    puts "The #{@color} #{@model} is driving."
  end

  def stop
    puts "The #{@color} #{@model} has stopped."
  end
end

my_car = Car.new("blue", "Honda")
my_car.drive # Output: The blue Honda is driving.
my_car.stop  # Output: The blue Honda has stopped.

Inheritance and Polymorphism

Inheritance allows you to create a new class based on an existing class. The new class inherits all the attributes and methods of the parent class:

# Inheriting from the Car class
class ElectricCar < Car
  def charge
    puts "The #{@color} #{@model} is charging."
  end
end

my_electric_car = ElectricCar.new("green", "Tesla")
my_electric_car.drive  # Output: The green Tesla is driving.
my_electric_car.charge # Output: The green Tesla is charging.

Encapsulation and Abstraction

Encapsulation keeps the data safe from outside interference and misuse. Abstraction helps reduce complexity by hiding unnecessary details. For example:

# Using private methods for encapsulation
class BankAccount
  def initialize(balance)
    @balance = balance
  end

  def deposit(amount)
    @balance += amount
    puts "Deposited #{amount}. New balance is #{@balance}."
  end

  def withdraw(amount)
    if amount <= @balance
      @balance -= amount
      puts "Withdrew #{amount}. New balance is #{@balance}."
    else
      puts "Insufficient funds!"
    end
  end
end

account = BankAccount.new(100)
account.deposit(50)  # Output: Deposited 50. New balance is 150.
account.withdraw(200) # Output: Insufficient funds!

Practical Examples of OOP

Let’s look at a practical example of using OOP to create a simple inventory system:

# Inventory system using OOP
class Product
  attr_accessor :name, :quantity

  def initialize(name, quantity)
    @name = name
    @quantity = quantity
  end

  def restock(amount)
    @quantity += amount
    puts "#{amount} of #{@name} restocked. New quantity: #{@quantity}."
  end
end

# Creating a product
product = Product.new("Laptop", 10)
product.restock(5) # Output: 5 of Laptop restocked. New quantity: 15.

Common Errors and Troubleshooting

When working with OOP, you may encounter common issues:

  • NoMethodError: This often occurs when trying to call a method that doesn't exist.
  • Instance Variable Errors: Ensure you use '@' for instance variables and access them within instance methods.
  • Inheritance Errors: Verify that child classes correctly inherit from parent classes.

Always read your error messages carefully, as they provide useful hints for troubleshooting.
