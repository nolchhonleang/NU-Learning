Setting up Git

Foundations Course

Introduction

Git is an incredibly powerful and widely-used version control system that helps developers keep track of changes in their code. Throughout your journey with NU-Learning, you will frequently encounter Git, so there’s no need to worry if you don’t fully grasp it at this very moment. The curriculum includes many lessons dedicated specifically to Git, where you will learn its functionality and best practices.

In addition, you will also become familiar with GitHub, a web-based platform that leverages Git, allowing you to upload, host, and manage your code with a user-friendly interface. It is a crucial tool for collaborating with other developers and showcasing your projects.

It's important to understand that while Git and GitHub are closely related and often used together, they are not the same entity. They serve different purposes and were developed by different organizations.

Step 1: Install Git

To get started, click on the operating system you are using from the options below:

Linux

Step 1.1: Update the system

Begin by running the following commands in your terminal to ensure that your Linux system is up to date:

sudo apt update
sudo apt upgrade

A note on typing passwords in the terminal

When you run commands in the terminal that require authentication (like sudo), you may notice that your password is not visible as you type it. This is completely normal! It’s a security feature designed to protect your password. Just type your password and press Enter, and everything will work as expected.

Step 1.2: Install Git

While you may already have Git installed, it's important to ensure you have the latest version. Execute the following commands to install or update Git:

sudo add-apt-repository ppa:git-core/ppa
sudo apt update
sudo apt install git

Step 1.3: Verify version

After installation, check that your version of Git is at least 2.28 by running this command:

git --version

If the version number is lower than 2.28, repeat the installation process to ensure you have the correct version.

MacOS

Step 1.0: Install Homebrew

First, you need to install Homebrew, which is a package manager for macOS. To install it, ensure that you meet the Homebrew MacOS requirements. Once you're ready, open your terminal and copy and paste the following command:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

If you are using an Apple Silicon Mac, there will be an additional step required after installation. Check the terminal output for “Installation Successful!” and follow the “Next steps” section to add Homebrew to your PATH, enabling the use of the brew command.

Step 1.1: Update Git

macOS typically comes pre-installed with a version of Git, but it is advisable to update to the latest version. Type the following command in your terminal:

brew install git

This command will install the latest version of Git. It's that simple!

Step 1.2: Verify version

After installing or updating Git, close your current terminal window and open a new terminal window. Check that your Git version is still at least 2.28 by running this command:

git --version

If the version number is below 2.28, go through the installation process again. If you encounter a no formulae found in taps error, follow these steps:

  1. Run brew doctor.
  2. Follow the commands provided by brew doctor to fix any issues.
  3. Finally, run brew install git, open a new terminal window, and check your Git version again.
ChromeOS

If you are using ChromeOS, you can refer to the instructions on installing Git from Source provided by Digital Ocean.

Step 2: Configure Git and GitHub

Step 2.1: Create a GitHub account

Navigate to GitHub.com and set up an account! During the registration process, you will be prompted to provide an email address. This email should be valid, as it will be used by GitHub to send you important notifications. Choose a strong password and create a username that reflects your identity as a developer. Follow the on-screen instructions to complete your registration.

Step 2.2: Link GitHub to Git

Once you have your GitHub account, it's time to connect it to your local Git installation. This allows you to synchronize your local repositories with those hosted on GitHub. Begin by running these commands:

git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"

Step 2.3: Test the configuration

To verify that your configurations are set up correctly, execute the command:

git config --list

This will display a list of your current configurations. Check to ensure your name and email are listed as expected.

Step 3: Learn the Basics of Git

Now that Git is installed and configured, it’s time to learn some fundamental Git commands. Below are some commands that you will use frequently:

  • git init: Initializes a new Git repository in your current directory.
  • git clone [repository URL]: Clones a repository from GitHub to your local machine.
  • git add [file]: Stages changes to be committed.
  • git commit -m "Your message": Commits the staged changes with a descriptive message.
  • git push: Pushes your local commits to the remote repository on GitHub.
  • git pull: Fetches changes from the remote repository and merges them into your local branch.

As you progress through your courses, you will practice these commands in real scenarios, helping you become more comfortable with version control.
