Websockets and Actioncable

Introduction

Welcome to the WebSockets and Action Cable module of the Ruby on Rails Course at NU-Learning! In this section, we will delve into the powerful features of WebSockets and how to use Action Cable to create real-time applications in Ruby on Rails.

By the end of this module, you will understand the concepts of WebSockets, how Action Cable integrates with Rails, and how to implement real-time features in your applications, such as chat systems, live notifications, and collaborative features.

Understanding WebSockets

WebSockets provide a full-duplex communication channel over a single TCP connection, allowing real-time data transfer between the client and server. This is particularly useful for applications that require constant updates, like chat applications or live dashboards.

Unlike traditional HTTP requests, which are stateless and require a new connection for each request, WebSockets maintain a persistent connection that allows both the server and client to send messages to each other at any time.

The WebSocket protocol begins with an HTTP handshake, after which the connection is upgraded to a WebSocket connection. Here’s a simple diagram to illustrate the flow:

Introduction to Action Cable

Action Cable is a framework that integrates WebSockets into Ruby on Rails applications, making it easy to handle real-time features without the need for complex setups.

Action Cable provides a simple interface to create channels that can broadcast messages to clients. Channels are similar to controllers in that they handle incoming connections and data, but they are specifically designed for real-time communication.

To start using Action Cable, you need to ensure it is included in your Rails application. Action Cable comes by default with Rails 5 and later versions.

Setting Up Action Cable

To set up Action Cable in your Rails application, follow these steps:

  1. Generate a new channel:
    2. rails generate channel Chat
  2. This will create a new channel file in app/channels/chat_channel.rb and a corresponding CoffeeScript file in app/javascript/channels/chat_channel.js.
  3. Define the channel in chat_channel.rb:
    4. class ChatChannel < ApplicationCable::Channel
              def subscribed
                stream_from "chat_channel"
              end
            
              def unsubscribed
                # Any cleanup needed when channel is unsubscribed
              end
            
              def speak(data)
                ActionCable.server.broadcast("chat_channel", message: data['message'])
              end
end
  4. In the JavaScript file, connect to the channel:
    5. import consumer from "./consumer"
            
            consumer.subscriptions.create("ChatChannel", {
              received(data) {
                // Append the received message to the chat window
                const messages = document.getElementById('messages');
                messages.innerHTML += 
                `${data.message}`;
              },
            
              speak(message) {
                this.perform('speak', { message: message });
              }
            });

Implementing Real-Time Features

With Action Cable set up, you can start implementing real-time features. For example, let’s create a simple chat application.

In your view file, add a form to send messages:

Now, add JavaScript to handle the form submission:

document.getElementById('new_message').addEventListener('submit', function(event) {
              event.preventDefault();
              const messageInput = document.getElementById('message');
              const message = messageInput.value;
              consumer.subscriptions.subscriptions[0].speak(message);
              messageInput.value = '';
            });

Now, when users submit messages, they will be sent through Action Cable and displayed in real-time in all connected clients.

Broadcasting Messages

Broadcasting messages is a key feature of Action Cable. You can send messages to all subscribers of a channel using the ActionCable.server.broadcast method. In the example above, when a user sends a message, it gets broadcast to all connected clients subscribed to the "chat_channel".

This allows for seamless communication and updates across all users without the need for manual refreshes or requests. You can also implement features like notifications by creating additional channels and broadcasting to specific users or groups.

Handling Disconnects

Handling disconnections is essential for maintaining a smooth user experience. In your channel, you can define what happens when a user disconnects:

        def unsubscribed
              # Notify other users that someone has left the chat
              ActionCable.server.broadcast("chat_channel", message: "#{current_user.name} has left the chat.")
            end

This allows you to inform other users about the status of participants in the chat.

Testing Action Cable

Testing Action Cable functionality is crucial to ensure that your real-time features work correctly. You can use RSpec and ActionCable's testing helpers to create tests for your channels.

Here’s an example of testing the ChatChannel:

require 'rails_helper'
            
            RSpec.describe ChatChannel, type: :channel do
              it 'subscribes to the chat channel' do
                subscribe
                expect(subscription).to be_confirmed
                expect(subscription).to have_stream_from('chat_channel')
              end
            
              it 'broadcasts a message to the chat channel' do
                subscribe
                perform :speak, message: 'Hello, World!'
                expect(ActionCable.server).to have_broadcasted_to('chat_channel').with(message: 'Hello, World!')
              end
            end

In these tests, we ensure that the channel subscribes correctly and that messages are broadcasted as expected.

Conclusion

With Action Cable, you can build interactive applications that provide users with live updates, notifications, and a seamless experience.
